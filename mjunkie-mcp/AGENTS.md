## Purpose

`mjunkie-mcp` is the **Sovereign Engine MCP server** for the Multimedia Junkie creative production ecosystem. It bridges the content pipeline database (`db.json`), brand configuration, widget-based UI templates, and autonomous persona workflows to any MCP-compatible AI agent framework (primarily Space Agent).

## Stable Contracts

### Brand color invariant
The primary brand color `#E91E63` **must** appear in:
- `HERO_BRANDING` widget gradient
- All `command_center` and `spatial_floating` layout brand sections
- Any HTML/CSS generated by Space Agent using this server

Never change the primary color without explicit user instruction. The `brand.ts` config is the single authoritative source.

### Tool naming convention
Tool names are `snake_case` verb-first: `query_content`, `update_beat_status`, `get_brand_guidelines`, `generate_dashboard_layout`. Do not rename existing tools without updating all callers (SKILL.md, index.ts, tests).

### Beat status enum
Valid beat status values: `needs_update` | `in_progress` | `completed` | `failed`. The enum is enforced in `updateBeatStatusTool.inputSchema`. Do not add values without updating the tool definition and tests.

### Dashboard modes
`generate_dashboard_layout` supports exactly two modes: `command_center` (12Ă—8 grid) and `spatial_floating` (3D depth-layered). New modes require a new handler function, a new enum value in the tool schema, and new tests.

### MCP Resource URIs
- `sovereign://db` â€“ Sovereign Engine database snapshot
- `brand://multimedia-junkie/guidelines` â€“ brand config JSON

Do not change these URIs without updating `SKILL.md` and any agent configurations referencing them.

## Tool Handler Pattern

All tool handlers follow this signature:
```typescript
export async function handleXxx(
  args: { ... },
  baseUrl = SOVEREIGN_ENGINE_BASE_URL,  // injectable for tests
): Promise<ToolResult>
```

The `baseUrl` parameter enables unit testing against a mock HTTP server without mocking `fetch` globally.

## Testing Rules

- Test files live in `mjunkie-mcp/tests/` named `phaseN.test.mjs`
- All async tool handlers are tested with a `createServer`-based mock instead of live services
- Run with: `npm test` (Node.js built-in test runner, no jest/vitest)
- Tests must pass without a running json-server or network access
